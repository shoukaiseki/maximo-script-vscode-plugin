### 1. Java Bridge 实现方式

**当前项目情况**：
- **本项目是纯 VSCode 扩展插件**，不是 Electron 应用
- 架构：`extension.ts` (入口) -> `completionProvider.ts` (补全逻辑) -> `configPanel.ts` (配置界面)
- 没有主进程（main.js）概念，无法直接调用 `node-java`

- **采用方案 B（预加载 JSON 缓存）**：已在 `completionProvider.ts` 中完整实现

- 实时反射层（第1层）已预留接口但注释掉，目前直接降级到第2层

**现有示例数据**：

**加载逻辑**：
```typescript
// completionProvider.ts:85-125
private async loadLocalApiData() {
  if (!this.config.localApiPath) {
    return;
  }

  try {
    const apiDir = this.config.localApiPath;
    
    if (!fs.existsSync(apiDir)) {
      console.warn(`API数据目录不存在: ${apiDir}`);
      return;
    }

    const files = fs.readdirSync(apiDir).filter(f => f.endsWith('.json'));
    
    for (const file of files) {
      try {
        const filePath = path.join(apiDir, file);
        const content = fs.readFileSync(filePath, 'utf-8');
        const data: ApiClassData = JSON.parse(content);
        
        // 缓存API数据，使用类名作为key
        this.localApiCache.set(data.className, data);
        
        // 也缓存简化的名称映射
        const simpleName = file.replace('.json', '').split('-').pop();
        if (simpleName) {
          this.localApiCache.set(simpleName, data);
        }
      } catch (error) {
        console.error(`加载API文件失败 ${file}:`, error);
      }
    }
    
    console.log(`已加载 ${this.localApiCache.size} 个API类数据`);
  } catch (error) {
    console.error('加载本地API数据失败:', error);
  }
}
```

**结论**：✅ 保持现有完整格式，代码已完美兼容，无需调整

### 3. 表达式映射的正则匹配性能

**实际代码实现**：
```typescript
// completionProvider.ts:462-523
private matchExpressionToType(
  expression: string,
  typeMap: Record<string, string>
): string | null {
  if (!expression || !typeMap) {
    return null;
  }
  
  console.log(`[ExpressionMatch] 🔍 尝试匹配表达式: "${expression}"`);
  
  // 第一步：尝试完全匹配（O(1) 查找）
  if (typeMap[expression]) {
    console.log(`[ExpressionMatch] ✅ 完全匹配成功: ${expression} -> ${typeMap[expression]}`);
    return typeMap[expression];
  }
  
  // 第二步：尝试正则匹配（仅当键名包含占位符时）
  for (const [key, type] of Object.entries(typeMap)) {
    if (key.includes('\\d') || key.includes('\\w') || key.includes('.*') || key.includes('\\s')) {
      // 将占位符转换为真正的正则表达式
      let pattern = key
        .replace(/\\\\d/g, '\\\\d+')
        .replace(/\\\\w/g, '\\\\w+')
        .replace(/\\.\\*/g, '.*')
        .replace(/\\\\s/g, '\\\\s+');
      
      // 转义其他特殊字符
      pattern = pattern.replace(/[.+?^${}()|[\\]\\\\]/g, (match) => {
        if (match === '\\\\' && pattern.substring(pattern.indexOf(match), pattern.indexOf(match) + 3).match(/\\\\[dws]/)) {
          return match;
        }
        return match === '.' ? '\\\\.' : match;
      });
      
      try {
        const regex = new RegExp(`^${pattern}$`);
        if (regex.test(expression)) {
          console.log(`[ExpressionMatch] ✅ 正则匹配成功: ${expression} ~ ${key} -> ${type}`);
          return type;
        }
      } catch (e) {
        console.warn(`[ExpressionMatch] ⚠️ 正则表达式构建失败: ${key}`, e);
      }
    }
  }
  
  // 第三步：返回值类型推断
  const methodCallMatch = expression.match(/\\.(\\w+)\\(/);
  if (methodCallMatch) {
    const methodName = methodCallMatch[1];
    if (this.methodReturnTypeMap[methodName]) {
      console.log(`[ExpressionMatch] ✅ 返回值类型推断: ${methodName} -> ${this.methodReturnTypeMap[methodName]}`);
      return this.methodReturnTypeMap[methodName];
    }
  }
  
  console.log(`[ExpressionMatch] ❌ 未找到匹配的类型`);
  return null;
}
```

**性能分析**：
1. **精确匹配优先**：O(1) 哈希表查找，最快路径
2. **正则匹配优化**：
   - 仅在键名包含占位符时才编译正则（通常 < 10 个条目）
   - JSDoc 解析产生的 typeMap 很小（用户自定义的表达式映射）
   - 不会像全局 API 数据那样有成千上万个条目
3. **快速失败**：如果前两步都没匹配，直接返回 null

**性能评估**：
- 精确匹配：O(1) - 微秒级
- 正则匹配：O(n)，但 n 通常 < 10 - 毫秒级
- 总体影响：可忽略不计（< 1ms）

**结论**：✅ 当前实现已足够高效，无需额外优化

**问题**：当前的 `parseJSDocTypes()` 只解析 JavaScript 风格的 JSDoc 注释，不支持 Python 类型注解。

**实现策略**：

**阶段 1（当前）**：仅实现 JavaScript 的 JSDoc 支持
- 理由：Maximo 脚本主要使用 JavaScript（Nashorn引擎）
- Python 支持可以后续扩展

**阶段 2（未来）**：添加 Python 类型注解解析器
```typescript
private parsePythonTypeHints(document: vscode.TextDocument): Record<string, string> {
  const typeMap: Record<string, string> = {};
  const text = document.getText();
  const lines = text.split('\n');
  
  // 匹配 Python 类型注解：varName: ClassName = ...
  const typeHintPattern = /^(\\w+)\\s*:\\s*([A-Za-z_][\\w.]*)\\s*=/;
  
  for (const line of lines) {
    const match = line.match(typeHintPattern);
    if (match) {
      const varName = match[1];
      const className = match[2];
      typeMap[varName] = className;
    }
  }
  
  return typeMap;
}
```

**结论**：⏸️ 第一阶段仅实现 JavaScript，Python 支持留待后续迭代
