## 背景
需要在现有插件中实现基于 JDK 反射和 JSDoc 类型注释的智能代码补全功能。

## 已分析现状
1. **现有架构**：
   - 已有基础的 CompletionProvider，支持本地 API JSON 数据缓存
   - 已有配置面板，包含服务器配置和补全设置
   - 支持 JavaScript 和 Python 语言的补全触发（`.` 和 `(`）

2. **已有功能**：
   - 从本地 JSON 文件加载 API 数据（reflection-data）
   - 内置常用对象补全（service、mbo、mboSet、userInfo）
   - 配置界面支持选择本地 API 数据目录

## 需要实现的核心功能（来自文档）
1. ✅ JSDoc 类型注释解析器
2. ✅ 隐式变量类型映射（mbo、mboset、service、userInfo）
3. ✅ 返回值类型推断（链式调用支持）
4. ✅ 表达式到类型的智能匹配（支持正则占位符 \d, \w, .*, \s）
5. ❓ 实时 JDK 反射（通过 Java Bridge）
6. ✅ 三层降级策略（实时反射 → JSON 缓存 → 常用 API）

## 疑问与待确认事项

### 1. Java Bridge 实现方式
**问题**：文档中提到使用 "node-java" 或 "javacall" 库建立 Node.js 与 JVM 的连接，但 VSCode 插件运行在 Node.js 环境中，无法直接嵌入 JVM。

**可能的方案**：
- **方案 A**：使用独立的 Java 进程作为桥接服务（通过 HTTP/gRPC 通信）
- **方案 B**：预先生成所有反射数据为 JSON 文件，不使用实时反射
- **方案 C**：使用 Electron 的主进程（如果是 Electron 应用）

**当前项目情况**：这是一个纯 VSCode 扩展插件，不是 Electron 应用，没有主进程概念。

**建议**：采用方案 B，完全依赖预加载的 JSON 数据，移除实时反射层。或者保留接口，未来可扩展。

---

### 2. reflection-data JSON 文件格式
**问题**：文档中的 JSON 示例格式与当前代码中的 ApiClassData 接口不完全一致。

**当前格式**（completionProvider.ts）：
```typescript
interface ApiClassData {
  className: string;
  superClass: string | null;
  interfaces: string[];
  methods: MethodInfo[];
}

interface MethodInfo {
  name: string;
  returnType: string;
  parameters: string[];
  description: string;
  modifiers: string;
  isStatic: boolean;
  isPublic: boolean;
  exceptions?: string[];
}
```

**文档格式**（JDK_REFLECTION_AND_JSDOC_COMPLETION.md）：
```json
{
  "className": "psdi.mbo.MboRemote",
  "methods": [
    {
      "name": "getString",
      "returnType": "java.lang.String",
      "parameters": ["java.lang.String"],
      "description": "Get string attribute value",
      "modifiers": "public abstract",
      "isStatic": false,
      "isPublic": true
    }
  ]
}
```

**问题**：
- 是否需要调整 JSON 文件格式？
- 还是保持现有格式，在代码中适配？

**建议**：保持现有格式更完整（包含 superClass、interfaces、exceptions），在代码中做兼容处理。

---

### 3. 表达式映射的正则匹配性能
**问题**：文档中的 `matchExpressionToType` 函数在每次补全时都会遍历所有 typeMap 进行正则匹配，如果 typeMap 很大可能影响性能。

**建议优化**：
- 缓存正则表达式对象，避免重复编译
- 限制正则匹配的尝试次数
- 优先使用精确匹配，正则匹配作为降级

---
